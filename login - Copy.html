Key Concepts

The Need for Information Security

The rapid growth of the internet and communications, coupled with the rise of Big Data and interconnected devices (IoT), has made security a critical subject. Early networks like ARPANET had minimal security, leading to vulnerabilities. Today's environment includes sophisticated criminals and complex threats targeting individuals, organizations, and critical infrastructure.

Examples of significant breaches illustrate the impact:

  • **Mirai Botnet (2016):** Used IoT devices for massive DDoS attacks.
  • **Ukraine Power Grid (2017):** Spear-phishing led to power outages affecting 1.4 million people.
  • **Amazon Ring (2019):** Hacked cameras invaded privacy.
  • **Oldsmar Water Supply (2021):** Attempted poisoning via system breach.
  • **Zyxel Gear (2023):** Exploited vulnerability used to build Mirai-like botnets.

The CIA Triad: Core Security Principles

These are the fundamental goals of information security:

  • Confidentiality: Preventing unauthorized disclosure of information. Ensuring only authorized individuals can access sensitive data.
  • Integrity: Ensuring data is accurate, complete, and trustworthy. Preventing unauthorized modification or deletion.
  • Availability: Ensuring systems and data are accessible and functional for authorized users when needed.

Security Attacks: Active vs. Passive

Attacks are actions compromising security.

Passive Attacks: Involve monitoring or eavesdropping without altering data. Goal is to obtain information.

  • Release of Message Content: Reading the content of communications.
  • Traffic Analysis: Observing patterns (who talks to whom, how often, how much) even if content is hidden.

Active Attacks: Involve modification of data streams or creation of false streams. Goal is to alter system resources or affect operation.

  • Masquerade: Pretending to be another entity.
  • Replay: Capturing and retransmitting valid data units.
  • Modification of Messages: Changing, deleting, or delaying legitimate messages.
  • Denial of Service (DoS): Preventing legitimate use of resources.

Passive attacks are hard to detect but easier to prevent (e.g., via encryption). Active attacks are harder to prevent but easier to detect (as they cause disruption).

Security Services and Mechanisms

Security Services: Aim to counter security attacks and provide desired security properties (often based on CIA + Authentication, Non-Repudiation). Examples: Confidentiality service, Integrity service, Authentication service.

Security Mechanisms: Specific methods or techniques used to implement security services.

  • Encipherment (Encryption): Provides confidentiality by making data unreadable without a key.
  • Data Integrity Mechanisms: Detect unauthorized data alteration (e.g., using hash functions, MACs).
  • Digital Signature: Provides authentication, integrity, and non-repudiation using asymmetric cryptography.
  • Access Control: Restricts access to resources based on identity/permissions (passwords, firewalls).
  • Traffic Padding: Hides traffic patterns by inserting dummy data.
  • Others include: Notarization, Routing Control.

The OSI Security Architecture provides a framework for classifying these attacks, services, and mechanisms.

Network Security Models

Models help visualize how security is applied in networks.

General Model: Shows Sender, Receiver, a secure Transformation (encryption/code), a Secret shared between them (key), an Information Channel (network), a potential Opponent on the channel, and often a Trusted Third Party (for key distribution).

Network Access Model: Shows computing resources (data, processes, software) protected by a 'Gatekeeper function' (like a firewall) controlling access from the network (Access channel). Opponents (hackers, malware) try to bypass the gatekeeper.

Key tasks in designing security services based on these models:

  1. Design a suitable transformation/encryption algorithm.
  2. Generate secret information (keys) for the algorithm.
  3. Develop secure methods for distributing/sharing keys.
  4. Specify protocols for using the algorithm and keys.

Threats in the Access Model:

  • Information Access Threats: Unauthorized interception or modification of data.
  • Service Threats: Exploiting flaws to deny service to legitimate users (e.g., viruses, worms).

Security Trinity: Prevention, Detection, Response

A balanced security posture relies on:

  • Prevention: Measures to stop attacks from succeeding (e.g., firewalls, strong passwords, encryption).
  • Detection: Measures to identify when attacks are occurring or have occurred (e.g., intrusion detection systems, log analysis).
  • Response: Actions taken to handle incidents, limit damage, and recover (e.g., incident response plans, backups, isolating systems).
